Научно-исследовательская работа на тему:
Разработка web-сайта с нуля

При верстке сайтов используются определенные языки: HTML5 – язык гипертекстовой разметки, CSS – каскадная таблица стилей, JavaScript–мультипарадигменный язык программирования, а также PHP, Python, Perl и т.п. Создание веб-страниц делится на 5 этапов.

1 Работа с макетом

Получив макет от дизайнера, нужно «вырезать» иконки, картинки и прочую графику. Также неплохо будет записать шрифты, которые используются в макете. Для этого можно использовать PhotoShop или GIMP, которые позволяют работать со слоями. После «вырезки» всей графики, ее нужно сложить в папку (обычно называют images). После всего можно приступать к следующему этапу – верстке.

2 Верстка

Для начала вам нужно создать два файла: html и css. В одном у вас будет содержание и разметка, в другом – стили. Очевидно, что верстать будет легче, если придерживаться порядка. Например, начать с прописывания базовых вещей в теге head. Здесь следует указать кодировку, путь к таблице стилей, title и т.д.


Далее начинается работа с самим содержимым. Это все, что будет находиться в теге body. Начинать верстать стоит с верхнего левого угла, двигаясь в правый нижний. То есть, сначала описать шапку сайта (слева на право), потом описать следующую структурную часть и таким образом постепенно переносить макет в код.


Сначала верстальщик пишет структуру в HTML, «вешая» элементам классы (class=”example”) и идентификаторы (id=”example”), чтобы позднее стилизовать их в CSS. Стоит придумывать адекватные имена классов (menu, head, widget)

3 Стилизация

После того, как структура написана, идентификаторы и классы заданы – можно начинать использовать CSS. Необязательно полностью прописывать структуру и только после этого начинать использовать CSS. Можно и даже нужно работать параллельно.


Дальнейший процесс верстки сайта в основном будет происходить с помощью CSS-файла, в котором прописываются нужные свойства и правила.


Стилизация должна выполняться по порядку. Сначала нужно прописать какие-то общие правила. Например, общий фон страницы, базовый шрифт, а также не забывать сбрасывать отступы.


Далее прописываются более специфические свойства – цвет рамок, стили для боковой колонки, футера (footer), расположение логотипа и многое другое. Грубо говоря, это второй этап стилизации, когда работа ведется с каждым отдельным блоком.

4 Скрипты

Верстальщик должен уметь работать с JavaScript-библиотеками. С их помощью задается различное поведение для веб-страниц. Пример: адаптивное меню. При уменьшении окна, допустим, до 500 пикселей, меню должно свернуться в одну кнопку. При клике на нее должен раскрываться вертикальный список (к примеру). Все это реализуется с помощью JavaScript.


Здесь нужно знать, какие задачи нам нужно выполнить с помощью скриптов. Если на сайте используется очень мало веб-сценариев, то нужно минимизировать код библиотек, чтобы не использовать огромные куски кода для реализации пары нужных вещей.

5 Проверка

После того, как верстка готова, остается проверить ее. Это можно делать с помощью открытия во всех браузерах и меняя размеры страниц (проверка на кроссбраузерность и адаптивность). Также проверить сайт можно с помощью сторонних сервисов, которые позволяют это сделать.
